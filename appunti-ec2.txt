  Perché 3 bandpass con 1 solo parametro freq?

  Guardando il codice:

  // Tutti e 3 i BPF usano la STESSA frequenza centrale!
  mBpf1L.setBandpass(freq, mSampleRate, resonance);

  mBpf2L.setBandpass(freq, mSampleRate, log(resProcess+1)/49.5f); // Q diverso
  mBpf3L.setBandpass(freq, mSampleRate, resonance);

  Il motivo storico (da EmissionControl2 originale):
  - I 3 filtri in cascata con la stessa frequenza ma Q diversi creano una curva di risonanza più "fat" e complessa
  - È un modo per ottenere risonanze più pronunciate senza instabilità
  - BPF2 ha un Q logaritmico intermedio per "ammorbidire" la cascata
  - Il mixing tra solo e cascade permette di interpolare da filtering gentile a risonanza estrema

  MA hai ragione: non è molto intuitivo e spreca CPU!

  Un singolo filtro non è abbastanza per alte risonanze (diventa instabile), quindi la cascata serve per stabilità. Ma è una soluzione "old school".

  2. Moog Ladder (4-pole Lowpass)

  Caratteristiche storiche:
  - Introdotto nel Moog Minimoog (1970)
  - 4 celle RC in cascata con feedback globale
  - Self-oscillation naturale a Q → ∞
  - Nonlinearità tanh per simulare saturazione transistor

  Implementazione da Faust:
  Assumo tu abbia qualcosa tipo il Zavalishin VA model o simile. Struttura tipica:

  Input → [LP1] → [LP2] → [LP3] → [LP4] → Output
            ↑_______________________________|
                      Feedback (Q)

  Parametri Moog:
  - freq: Cutoff frequency (Hz)
  - resonance: 0.0-1.0 → feedback amount (0-4.0 tipicamente)
  - Self-oscillation quando resonance ≈ 1.0

  Vantaggi:
  - ✅ Suono "vintage synth" classico
  - ✅ Risonanze musicali e stabili
  - ✅ Comportamento predicibile

  3. Highpass Virtual Analog - Quale usare?

  Qui hai diverse opzioni storiche. Ecco le migliori:

  Opzione A: Steiner-Parker (Synthi AKS) - LA MIA RACCOMANDAZIONE ⭐

  Storia:
  - Progettato da Nyle Steiner per EMS Synthi AKS (1971)
  - Usato da Brian Eno, Pink Floyd
  - Filtro state-variable multi-mode (LP/BP/HP simultanei)

  Perché è perfetto:
  - ✅ Highpass E bandpass nella stessa struttura (efficiente!)
  - ✅ Risonanza stabile fino a self-oscillation
  - ✅ Suono "acido, metallico" diverso dal Moog
  - ✅ VA implementation ben documentata (Zavalishin, Pirkle)

  Topologia:
  Input → Integrator1 → Integrator2
           ↓ (BP)        ↓ (LP)
           └─────────────┴→ Feedback → HP output

  Coefficienti per HP output:
  hp = input - bp * resonance - lp
  // dove bp e lp sono output degli integratori

  Vantaggi vs Moog per HP:
  - Più "aggressivo" e "risonante" (carattere diverso)
  - Self-oscillation più pronunciata
  - Curva Q più "spigolosa" (vintage)

  ---
  Opzione B: Oberheim SEM (2-pole)

  Storia:
  - Tom Oberheim, 1974
  - Filtro state-variable con morph continuo LP→BP→HP

  Caratteristiche:
  - 2-pole (12dB/oct)
  - Risonanza "soft clipping" per calore analogico
  - Suono "smooth, musicale"

  Pro/Contro:
  - ✅ Più dolce del Steiner
  - ✅ CPU leggero (2-pole)
  - ⚠️ Meno carattere distintivo

  ---
  Opzione C: MS-20 Sallen-Key (alternativa esotica)

  Storia:
  - Korg MS-20 (1978)
  - Topologia Sallen-Key con distorsione pesante

  Caratteristiche:
  - Highpass molto risonante
  - Distorsione integrata (diodi clipper)
  - Suono "screaming, harsh"

  Pro/Contro:
  - ✅ Carattere MOLTO distintivo (aggressivo)
  - ⚠️ Forse troppo colorato per granular general-purpose
  - ⚠️ Più complesso da implementare

  La Mia Raccomandazione Finale

  Implementa 3 filter types:

  | Type         | Implementazione                      | Carattere                   | CPU     | Storico         |
  |--------------|--------------------------------------|-----------------------------|---------|-----------------|
  | 0 - Bandpass | Biquad 2-pole singolo (semplificato) | Neutro, trasparente         | Leggero | RBJ cookbook    |
  | 1 - Lowpass  | Moog Ladder 4-pole VA                | Caldo, grasso, synth        | Medio   | Minimoog 1970   |
  | 2 - Highpass | Steiner-Parker SVF                   | Acido, metallico, risonante | Medio   | Synthi AKS 1971 |

  Perché Steiner-Parker per HP?
  1. ✅ Complementare al Moog: dove Moog è caldo, Steiner è freddo/metallico
  2. ✅ Stesso "peso" storico: entrambi filtri iconici degli anni '70
  3. ✅ Risonanza simile: self-oscillation controllabile come Moog
  4. ✅ Efficienza: implementazione SVF è quasi identica in complessità al Moog
  5. ✅ Versatilità: SVF dà anche BP gratis (puoi esporre 4 modi se vuoi)

  enum FilterType {
    FILTER_BANDPASS = 0,  // Biquad BP (attuale semplificato)
    FILTER_MOOG = 1,      // Moog Ladder 4-pole LP
    FILTER_STEINER = 2    // Steiner-Parker SVF HP
  };

  class GrainFilter {
    FilterType mType;

    // Implementazioni
    Biquad mBiquad;           // Per bandpass
    MoogLadder mMoog;         // Per lowpass
    SteinerSVF mSteiner;      // Per highpass

    float process(float input) {
      switch(mType) {
        case FILTER_BANDPASS: return mBiquad(input);
        case FILTER_MOOG: return mMoog(input);
        case FILTER_STEINER: return mSteiner.getHP(input);
      }
    }
  };
